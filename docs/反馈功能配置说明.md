# 反馈功能配置说明

## 功能概述

用户可以通过"使用反馈"页面提交文字反馈和截图，系统会自动发送邮件到 `liuguangyu@kanyun.com`。

## 配置步骤

### 1. 安装依赖包

已添加到项目中:
- `nodemailer` - 用于发送邮件
- `multer` - 用于处理文件上传

如果 npm 安装失败,请手动执行:
```bash
cd web
npm install nodemailer multer
```

### 2. 配置 SMTP 邮箱

需要在 `web/.env` 文件中配置 SMTP 服务器信息。推荐使用以下邮箱服务:

#### 方案 A: QQ 邮箱 (推荐)

**获取授权码步骤:**
1. 登录 QQ 邮箱网页版
2. 点击"设置" -> "账户"
3. 找到"POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV服务"
4. 开启"IMAP/SMTP服务"或"POP3/SMTP服务"
5. 点击"生成授权码",按照提示发送短信
6. 记录生成的授权码(16位字符)

**配置 .env:**
```env
SMTP_HOST=smtp.qq.com
SMTP_PORT=587
SMTP_USER=你的QQ邮箱@qq.com
SMTP_PASS=刚才获取的16位授权码
```

#### 方案 B: 163 邮箱

**获取授权码步骤:**
1. 登录 163 邮箱网页版
2. 设置 -> POP3/SMTP/IMAP
3. 开启"IMAP/SMTP服务"
4. 设置客户端授权密码

**配置 .env:**
```env
SMTP_HOST=smtp.163.com
SMTP_PORT=465
SMTP_USER=你的163邮箱@163.com
SMTP_PASS=客户端授权密码
```

#### 方案 C: Gmail (需科学上网)

**配置 .env:**
```env
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=你的Gmail地址@gmail.com
SMTP_PASS=应用专用密码
```

**注意**: Gmail 需要开启"两步验证"并生成"应用专用密码"

### 3. 重启服务器

配置完成后,重启 Node.js 服务器:
```bash
# 停止旧进程
ps aux | grep server.js
kill <进程ID>

# 启动新服务
cd web
npm run dev
```

或者使用启动脚本:
```bash
cd web
./start_server.sh
```

## 功能测试

## 故障排查

### 问题1: 提交失败,显示"提交失败,请稍后重试"

**可能原因:**
- SMTP 配置错误
- 授权码不正确
- 网络问题

**解决方法:**
1. 检查服务器控制台的错误信息:
   ```bash
   # 查看服务器日志
   tail -f /path/to/server.log
   ```

2. 验证 .env 配置:
   - SMTP_HOST 是否正确
   - SMTP_PORT 是否正确
   - SMTP_USER 是否包含完整邮箱地址
   - SMTP_PASS 是否是授权码(不是邮箱密码)

3. 测试 SMTP 连接:
   ```javascript
   // 在 server.js 中临时添加测试代码
   transporter.verify((error, success) => {
     if (error) {
       console.log('SMTP连接失败:', error);
     } else {
       console.log('SMTP连接成功');
     }
   });
   ```

### 问题2: npm install 失败

**解决方法:**
```bash
# 清除缓存
npm cache clean --force

# 切换npm镜像
npm config set registry https://registry.npmmirror.com

# 重新安装
npm install nodemailer multer
```

### 问题3: 图片上传失败

**可能原因:**
- 图片大小超过 10MB
- 图片格式不支持

**解决方法:**
- 压缩图片后重试
- 确保图片格式为 PNG/JPG/GIF

## 安全注意事项

1. **不要将 .env 文件提交到 git**
   - 已在 .gitignore 中排除
   - 授权码是敏感信息,泄露后需立即重置

2. **定期更换授权码**
   - 建议每3-6个月更换一次SMTP授权码

3. **监控反馈提交频率**
   - 如有异常高频提交,可能是被滥用
   - 可在服务器端添加频率限制
