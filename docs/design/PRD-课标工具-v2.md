# 课标工具 · 产品需求文档（PRD）

> 版本：v2.0 | 日期：2025-02-14
> 定位：内部效率工具，供课程设计团队使用（<10人）

---

## 一、产品定义

### 1.1 一句话定义

将分散在多份课标文件中的要求，按用户关心的维度即时重组，并支持 AI 对重组结果进行分析与内容生成。

### 1.2 解决的核心问题

课程设计者每次工作时，需要从多份课标文件（不同国家、不同学科、不同年级）中手动检索、汇总、比较相关条目，再将这些条目作为上下文交给 AI 分析。这个"跨文件上下文组装"过程重复、耗时、易遗漏。

### 1.3 两条核心使用路径

| | 路径1：课标 → 内容 | 路径2：内容 → 课标 |
|---|---|---|
| 起点 | 课标条目 | 知识/活动描述 |
| 过程 | 选中课标 → AI 基于课标生成内容建议 | 输入描述 → AI 在课标库中语义匹配 |
| 终点 | 获得知识/活动/比较分析建议 | 看到对应课标及年级，判断难度与必要性 |
| 是否需要 AI 多轮对话 | 是 | 通常不需要 |

两条路径共享课标浏览器作为起点。路径2在浏览器内闭环，路径1从浏览器延伸到 AI 工作台。

### 1.4 用户画像

- 课程设计师，日常工作涉及多国课标的查阅与比较
- 熟悉课标结构，能准确判断 AI 输出质量
- 典型任务：基于若干课标条目寻找相关知识/活动建议，或反向确认某个知识点对应哪些课标要求

### 1.5 不做什么

- 不做课程方案生成（不输出完整教案）
- 不做课标数据的在线编辑/协作管理（数据通过 JSON 文件离线维护）
- 不做任何社交/分享/评论功能
- 不做用户体系/权限管理（内部工具，无需登录）

## 二、数据基础

### 2.1 课标数据源

| 数据源 | 子文件 | 状态 | 格式 |
|--------|--------|------|------|
| NGSS-DCI | DCI（学科核心概念） | ✅ 已有 | JSON |
| NGSS-PE | PE（表现期望） | ✅ 已有 | JSON |
| NGSS-SEP | SEP（科学与工程实践） | 🔄 完善中 | JSON |
| NGSS-CCC | CCC（跨学科概念） | 🔄 完善中 | JSON |
| 中国义务教育科学课标 | 义教科学课标 | ✅ 已有 | JSON |
| 中国高中课标 | 中国高中课标 | ⏳ 待整理 | PDF → JSON |

### 2.2 数据模型要求

每条课标条目必须包含以下结构化字段：

```json
{
  "id": "唯一标识",
  "source": "课标来源，如 NGSS-DCI / NGSS-PE / NGSS-SEP / NGSS-CCC / 中国义务教育科学课标 / 中国高中课标",
  "document": "子文件，如 DCI / PE / SEP / CCC / 义教科学课标 / 中国高中课标",
  "domain": "领域，如 Physical Science / 物质科学",
  "topic": "主题/大概念，如 Forces and Motion / 力与运动",
  "grade_band": "年级段，统一为 K / 1-2 / 3-4 / 5-6 / MS / HS",
  "text": "课标原文",
  "text_en": "英文原文（如有）",
  "core_concept_title": "核心概念标题（英文）",
  "core_concept_title_cn": "核心概念标题（中文）",
  "sub_concept_title": "子概念标题（英文）",
  "sub_concept_title_cn": "子概念标题（中文）"
}
```

说明：
- `grade_band` 统一为标准年级段：K（幼儿园）、1-2、3-4、5-6、MS（初中）、HS（高中）
- `core_concept_title` 和 `sub_concept_title` 字段用于层级化展示课标概念体系
- 不同课标的领域命名体系保留原始表述，不做强制统一
- 数据通过导入 JSON 文件更新，工具本身不提供数据编辑界面

## 三、产品结构

### 3.1 整体架构

```
底座层：课标数据库（JSON → 结构化索引 + 语义索引，支持关键词搜索与语义匹配）

界面A：课标浏览器
  ├── 模式1：关键词搜索（筛选 + 关键词 → 结构化检索）
  └── 模式2：智能匹配（自然语言描述 → AI 语义匹配 + 匹配理由）

界面B：AI 工作台（基于选中课标条目进行分析/生成）
```

两个主界面 + 浏览器内两个搜索模式。

路径2（内容 → 课标）在浏览器的智能匹配模式内完成闭环。
路径1（课标 → 内容）从浏览器任一模式出发，勾选条目后进入工作台。

---

### 3.2 课标浏览器（界面A）

浏览器包含两个搜索模式，通过搜索框下方的模式切换标签切换。两个模式共享同一个筛选面板、结果列表区域和底部选中栏。

#### 3.2.1 公共组件

**筛选面板（左侧，常驻）**
- 维度：课标来源（source）、领域（domain）、年级（grade_band）
- 每个维度多选（复选框），结果取并集
- 联动：筛选项显示当前结果中的可选项及数量，避免空结果
- 筛选在两个搜索模式中均可用，可叠加使用

**结果列表（右侧）**
- 表格行形式，列：复选框 | 来源标签 | 年级 | 主题 | 课标原文
- 每条左侧有复选框，支持多选
- 已勾选条目在搜索/筛选/模式切换后保持选中状态

**底部固定栏**
- 内容：已选数量徽章 + "N 条课标已选中" + "进入工作台 →" 按钮
- 未选中时：徽章灰色，按钮半透明但仍可点击（进入工作台后为空状态）
- 有选中时：徽章蓝色，按钮正常激活
- 在两个搜索模式中行为一致

#### 3.2.2 模式1：关键词搜索

**搜索框**
- 单行输入框，页面顶部
- placeholder："输入关键词搜索课标..."
- 实时过滤：输入后即时匹配 `text` + `keywords` + `topic` 字段
- 搜索与筛选可叠加

**结果展示**
- 标准表格行，无额外信息

#### 3.2.3 模式2：智能匹配

**输入区**
- 多行文本输入框（textarea），可粘贴较长的知识/活动描述
- placeholder："描述你想讲的知识或已设计的活动，AI 将匹配对应课标..."
- 右侧"匹配课标"按钮，点击后触发 AI 语义匹配
- 非实时：需要用户主动点击按钮触发（区别于关键词搜索的实时过滤）

**结果展示**
- 顶部绿色横幅：显示匹配概况，如"为「弹簧秤测量重力...」匹配到 4 条相关课标"
- 每条结果在课标原文下方额外显示一行匹配理由（绿色小字），由 AI 生成
- 匹配理由需要说明：
  - 为什么该课标与输入内容相关
  - 如果存在不完全匹配（如难度超出/低于该年级要求），需明确指出
- 结果行带绿色左边框，与关键词搜索的蓝色选中态视觉区分
- 筛选面板同样可用，对匹配结果进一步过滤

**精度要求**
- 条目级别匹配（如"NGSS PE 3-5, Forces and Interactions"）
- 按相关度排序

#### 3.2.4 不做

- 不做条目排序（课标条目无排序意义；智能匹配结果按相关度排序由 AI 决定）
- 不做分页（数据量可控，虚拟滚动）
- 不做保存/收藏（首版不做）

---

### 3.3 AI 工作台（界面B）

#### 3.3.1 布局

```
左侧面板（约 30-35%）：已选课标条目
右侧区域（约 65-70%）：AI 对话
```

#### 3.3.2 左侧：已选条目面板

- 标题："当前课标上下文" + 条目计数
- 每条条目为小卡片：来源标签 + 年级 + 主题 + 原文摘要 + ✕ 移除按钮
- 底部："+ 添加更多条目"按钮，点击切回浏览器，工作台对话状态保留
- 条目变化时，AI 后续回答自动基于更新后的条目集
- 条目为空时显示引导文字

#### 3.3.3 右侧：AI 对话区

**意图栏**
- 位于对话区顶部
- 4 个意图按钮，全部基于左侧已选条目工作：

| 意图 | 触发条件 | 说明 |
|------|----------|------|
| 比较 | 选中 ≥ 2 条时可用 | 跨课标/跨年级的要求对比，找共性与差异 |
| 提炼 | 选中 ≥ 2 条时可用 | 从多条课标中抽象出上位概念或能力进阶线索 |
| 找知识 | 选中 ≥ 1 条时可用 | 基于课标要求，推荐相关科普知识、趣味事实、冷知识 |
| 找活动 | 选中 ≥ 1 条时可用 | 基于课标要求，推荐实验设计、探究活动、线上互动活动 |

- 右侧灰色提示文字："基于左侧 N 条课标条目"
- 点击意图按钮 → 输入框自动填充预设 prompt → 用户可编辑后发送
- "匹配课标"不在工作台中出现，该功能已由浏览器的智能匹配模式承担

**对话内容区**
- 标准聊天气泡布局
- AI 回复支持 Markdown 渲染
- AI 每次回答时，系统 prompt 中自动注入当前选中的所有课标条目原文作为上下文

**输入框**
- placeholder："基于已选课标继续提问..."
- 支持自由输入 + 回车发送

#### 3.3.4 不做

- 不做对话历史保存（首版不做）
- 不做 AI 输出的导出/下载（首版不做，用户可复制文本）
- 不做 AI 模型切换
- 不做课标匹配功能（已移至浏览器）

## 四、核心用户路径

### 路径1：课标 → 内容（最高频）

```
打开工具
  → 浏览器（关键词搜索模式）
  → 输入关键词 / 选择筛选条件 / 两者结合
  → 在结果列表中勾选若干条目
  → 点击"进入工作台"
  → 查看已选条目，点击意图按钮（比较 / 提炼 / 找知识 / 找活动）
  → 获得 AI 分析结果
  → （可选）追问 / 修改条目 / 切换意图
```

最短路径：3 步操作（筛选 → 勾选 → 进入工作台 → 点击意图）。

### 路径2：内容 → 课标（在浏览器内闭环）

```
打开工具
  → 浏览器，切换到"智能匹配"模式
  → 输入知识/活动描述，点击"匹配课标"
  → 查看匹配结果（课标条目 + 匹配理由）
  → 判断该知识的难度、对应年级、是否必须讲
  → 完成
```

路径2也可衔接路径1：
```
  → （可选）勾选匹配到的部分条目
  → 点击"进入工作台"
  → 继续使用路径1的工作流
```

## 五、技术约束与建议

### 5.1 数据层

- 课标数据以 JSON 文件存储，启动时加载到内存
- 关键词搜索：需建立全文搜索索引（Fuse.js、MiniSearch 或 SQLite FTS）
- 智能匹配：需要语义匹配能力，两种实现路径：
  - 方案A：课标条目做 embedding 预计算，用户输入做 embedding，计算相似度 → 返回 top-N → 再调用 LLM 生成匹配理由
  - 方案B（数据量可控时更简单）：将全量课标摘要作为上下文传入 LLM，让 AI 直接返回匹配结果 + 理由

### 5.2 AI 集成

- AI 工作台：选中课标条目原文 + 用户指令组装成 prompt，调用 LLM API
- 智能匹配：用户描述 + 课标数据库 → LLM 返回匹配结果 + 理由
- 意图按钮对应预设 prompt 模板，模板可配置

### 5.3 部署

- 内部工具，本地 Web 应用或内网服务
- 无需高并发
- 数据更新通过替换 JSON 文件 + 重启/热加载

## 六、版本规划建议

### v1.0（核心可用）

- 课标浏览器：关键词搜索 + 筛选 + 多选
- AI 工作台：条目面板 + 4 个意图 + 自由对话
- 支持已有 JSON 数据源

### v1.1（智能匹配）

- 浏览器新增智能匹配模式
- AI 语义匹配 + 匹配理由生成
- 匹配结果可勾选后进入工作台

### v1.2（体验优化）

- 对话历史保存与回溯
- AI 输出导出为 Markdown
- 浏览器默认恢复上次筛选状态

### v1.3（数据扩展）

- 支持新课标 JSON 导入后自动建立索引
- 课标条目 embedding 预计算，提升匹配精度

## 七、交互规格补充

### 搜索与筛选联动
- 搜索和筛选可叠加使用，互不排斥
- 修改搜索词或筛选条件时，结果实时更新
- 已勾选的条目在搜索/筛选/模式切换后保持选中状态

### 模式切换
- 关键词搜索 ↔ 智能匹配通过搜索框下方标签切换
- 切换时保留已勾选的条目
- 切换时清空搜索/匹配内容（两个模式的输入互不干扰）

### 浏览器 ↔ 工作台切换
- 从浏览器任一模式勾选条目后，点击"进入工作台"将条目带入左侧面板
- 从工作台点击"添加更多条目"切回浏览器，对话状态保留
- 添加/移除条目后回到工作台，对话继续，AI 后续回答基于更新后的条目集

## 附录：预设 Prompt 模板参考

### 比较

```
以下是用户选中的多条课标要求，请从以下维度进行对比分析：
1. 各条课标对同一主题的要求有何共性？
2. 各条课标的要求有何关键差异？（深度、广度、侧重点）
3. 这些差异背后可能反映了怎样的教育理念差异？

课标条目：
{selected_items}
```

### 提炼

```
以下是用户选中的多条课标要求，请提炼：
1. 这些课标条目共同指向的上位科学概念是什么？
2. 从低年级到高年级，能力要求的进阶线索是什么？
3. 用一句话概括这些课标的核心教学目标。

课标条目：
{selected_items}
```

### 找知识

```
以下是用户选中的课标要求。请基于这些要求，推荐尽可能多的相关内容：
- 科普知识（教材不讲但有趣的）
- 冷知识 / 趣味事实
- 前沿科学进展
- 日常生活中的应用实例
- 跨学科关联知识

要求：越多越好，越有趣越好。每条简要说明与课标的关联。

课标条目：
{selected_items}
```

### 找活动

```
以下是用户选中的课标要求。请基于这些要求，推荐可行的活动设计：
- 线上探究活动
- 简易实验设计（可在家/教室完成）
- 互动模拟或思维实验
- 讨论/辩论话题

每个活动请说明：活动名称、简要流程、对应的课标要求、适合年级。

课标条目：
{selected_items}
```

### 智能匹配（浏览器使用）

```
用户描述了一个知识点或教学活动，请在以下课标数据库中找到所有相关的课标条目。

要求：
- 精确到条目级别
- 返回格式：课标来源 | 领域 | 年级 | 原文 | 匹配理由
- 匹配理由需说明该课标与用户描述的关联点
- 如果用户描述的内容难度超出或低于某课标的年级要求，需在匹配理由中明确指出
- 按相关度排序

用户输入：
{user_input}

课标数据库：
{all_standards_summary}
```
